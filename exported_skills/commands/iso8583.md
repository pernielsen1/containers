## What this project is

A Python-based test client/server for ISO 8583 authorization messages. The client reads test cases from a CSV, sends them as ISO 8583 messages over TCP, and records responses. The server echoes key fields and approves/declines based on a PAN prefix whitelist.

- Library: `pyiso8583` (`import iso8583`)
- Python: `python3`
- Working directory: `/home/perni/containers/claude_exp/auth_test/`

## Spec Files

Each field entry has:
```json
{
  "data_enc": "ascii",   // or "cp500" (EBCDIC) or "b" (binary)
  "len_enc":  "ascii",   // or "cp500" or "b"
  "len_type": 0,         // 0=fixed, 2=LLVAR, 3=LLLVAR
  "max_len":  6,
  "desc":     "Processing Code"
}
```

- `test_spec.json`: all character fields use `"ascii"`, bitmaps/PIN/MAC use `"b"`
- `mc_spec.json`: all character fields use `"cp500"` (EBCDIC cp500), bitmaps/PIN/MAC/ICC use `"b"`; follows MasterCard authorization field definitions

To select a spec per role, set `iso_spec` in `config.json` under `"client"` or `"server"`.

## TCP Framing Schemes

| Scheme | Wire format |
|--------|------------|
| `TCP_framing_standard` | 4-byte big-endian uint32 length + message bytes |
| `TCP_framing_FFFF_nnnn` | `0xFFFFFFFF` marker + 4 ASCII digit length + message bytes |

Set `tcp_framing` in `config.json`, or override with `--framing` on the CLI.

## Input CSV (`input/test_cases.csv`)

- Separator: **semicolon** (`;`), encoding: **`utf-8-sig`**
- Column names are ISO 8583 field numbers: `2`, `3`, `4`, `11`, `37`, `41`, `42`, `49`, `63`, etc.
- `expected_39`: expected response code — used for mismatch detection
- Field `63` is used as the correlation key between request and response
- **Fixed-length fields** (`len_type: 0`): write the value as a plain number (e.g. `0`); the client zero-pads left to `max_len` automatically
- Variable-length fields: write the value as-is; no padding applied
- Binary fields (`data_enc: "b"`) are excluded from CSV input

---

## Key Implementation Details

- **Zero-padding**: in `send_thread`, any fixed-length non-binary field (`len_type == 0`, `data_enc != "b"`) is left-padded with zeroes to `max_len` before encoding. This allows writing `0` in the CSV for a 6-digit field like field 3.
- **STAN** (field 11) is auto-generated as a 6-digit zero-padded sequence; overrides any value in the CSV.
- **Correlation**: field `63` from the request row is used to match responses; must be unique per test run.
- **Batch control**: `batch_size` messages sent, then pause `batch_wait` seconds; controlled via `config.json`.
- **Server approval logic**: server checks PAN (field 2) against `positive_list.json` prefixes → `39=00` (approved) or `39=01` (declined).
- **Auto fields** (never taken from CSV): `h`, `p`, `1`, and all binary fields.
